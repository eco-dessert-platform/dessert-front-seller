코딩 표준, API 레퍼런스 등 개발 시 참조할 수 있는 자료입니다.

## 📑 목차

### 코딩 표준

프로젝트에서 사용하는 코딩 표준과 컨벤션입니다.

#### [액션 명명 규칙](./coding-standards/action-naming.md)

**주요 내용**:

- 기본 명명 규칙
- 비동기 액션 명명
- 로컬 리듀서 주의사항

#### [코드 스타일 가이드](./coding-standards/code-style.md)

Prettier와 ESLint를 사용한 코드 포맷팅 규칙입니다.

**주요 내용**:

- Prettier 설정
- IDE별 자동 포맷팅 설정
- TailwindCSS 클래스 정렬

#### [Git 커밋 규칙](./coding-standards/git-commit.md)

Git 커밋 메시지 작성 규칙입니다.

**주요 내용**:

- 커밋 메시지 형식
- Type 종류
- 예시 및 금지 사례

### API 레퍼런스

프로젝트에서 제공하는 유틸리티 API 문서입니다.

reduxMaker 및 관련 유틸리티 함수의 상세 API 문서입니다.

**주요 내용**:

- reduxMaker 함수 시그니처

- reducerUtils 함수들
- 사용 예제

## 레퍼런스 활용 방법

### 코딩 표준

- 새로운 기능 개발 전에 관련 표준을 확인하세요
- 코드 리뷰 시 기준으로 활용하세요
- 팀 온보딩 자료로 사용하세요

### API 레퍼런스

- 함수 사용법이 궁금할 때 참조하세요
- 타입 정의를 확인할 때 사용하세요
- 예제 코드를 복사하여 활용하세요
